# 定时点击助手 v0.6.0 - 多时间点任务系统

## 🎉 重大更新

成功实现**多时间点任务系统**，现在一个任务可以在不同时间点执行不同操作，完美解决复杂自动化场景！

---

## 新增功能

### 1. 多时间点任务 ⭐

**核心能力**：
- 一个任务包含多个时间点
- 每个时间点独立执行
- 支持不同点击序列和间隔

**典型场景**：
```
20:57 → 进入小程序（回到桌面 → 打开微信 → 进入小程序）
21:00 → 刷新抢课（点击刷新 → 选课 → 确认）
```

**配置示例**：
```json
{
  "name": "小程序抢课",
  "type": "multi_time",
  "timeline": [
    {
      "time": "20:57",
      "name": "进入小程序",
      "show_desktop": true,
      "clicks": [[100, 600], [500, 300]],
      "interval": 2.0
    },
    {
      "time": "21:00",
      "name": "刷新抢课",
      "clicks": [[800, 300], [850, 400]],
      "interval": 0.3
    }
  ]
}
```

### 2. 自动回到桌面 🖥️

**功能**：
- 支持 `show_desktop: true` 选项
- 自动按 `Win+D` 回到桌面
- 适合从桌面图标开始的操作

**使用场景**：
- 打开桌面应用
- 从任务栏启动程序
- 确保干净的起始状态

### 3. 独立间隔配置 ⏱️

**灵活性**：
- 每个时间点可设置不同间隔
- 准备阶段：2 秒（等待加载）
- 抢购阶段：0.3 秒（快速点击）

### 升级模块

**1. ConfigManager (`src/config_manager.py`)**
- 支持 `type: "simple"` 和 `type: "multi_time"`

**3. Clicker (`src/clicker.py`)**

- 更新 `/api/tasks` POST 接口
- 支持多种任务类型

## 配置参数说明

### Timeline 对象

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `time` | string | ✅ | - | 执行时间 `HH:MM` |
| `name` | string | ✅ | - | 步骤名称 |
| `clicks` | array | ✅ | - | 坐标列表 `[[x,y]]` |
| `interval` | float | ❌ | 0.5 | 点击间隔（秒）|
| `show_desktop` | boolean | ❌ | false | 是否先回到桌面 |

### 任务类型

- `"simple"` - 简单任务（单一时间，向后兼容）
- `"multi_time"` - 多时间点任务（新功能）

## 常见问题

**Q: Web 界面支持创建多时间点任务吗？**

A: 当前版本（v0.6.0）暂不支持，请使用：
- 创建工具：`python create_multi_time_task.py`
- 手动编辑：`config/tasks.json`
- API 调用

Web 界面可视化编辑器将在后续版本实现。

**Q: 能同时运行多个多时间点任务吗？**

A: 可以！调度器会自动处理所有任务的所有时间点。

**Q: 时间点失败会影响后续时间点吗？**

A: 不会。每个时间点独立执行，互不影响。

**Q: 如何调试坐标？**

A: 建议方法：
1. 创建测试任务，时间设为 1-2 分钟后
2. 逐个测试每个时间点
3. 确认无误后修改为实际时间
