# 定时点击助手 v0.7.0 升级说明

## 🎉 新增功能：键盘操作支持

### 核心改进

在 v0.7.0 版本中，我们为"多时间点任务"添加了完整的键盘操作支持，现在可以混合使用鼠标点击和键盘操作！

### 支持的操作类型

#### 1. 点击操作（Click）
- 单击/双击/多次点击指定坐标
- 可设置点击次数（1-3次）
- 支持获取当前鼠标位置

**示例：**
```json
{
  "type": "click",
  "x": 960,
  "y": 540,
  "clicks": 2,
  "label": "双击登录按钮"
}
```

#### 2. 文本输入（Type）
- 输入任意文本内容
- 可自定义字符输入间隔
- 适用于账号密码、搜索关键词等场景

**示例：**
```json
{
  "type": "type",
  "text": "my_username",
  "interval": 0.05,
  "label": "输入用户名"
}
```

#### 3. 按键/快捷键（Hotkey）
- 单个按键：Enter、Tab、Esc、Space 等
- 组合快捷键：Ctrl+C、Ctrl+V、Alt+Tab、Win+D 等
- 支持所有标准键盘按键

**常用快捷键示例：**
- `enter` - 回车确认
- `tab` - 切换输入框
- `ctrl+c` - 复制
- `ctrl+v` - 粘贴
- `ctrl+a` - 全选
- `alt+tab` - 切换窗口
- `win+d` - 显示桌面
- `win+r` - 打开运行对话框

**示例：**
```json
{
  "type": "hotkey",
  "keys": ["ctrl", "c"],
  "label": "复制文本"
}
```

#### 4. 等待操作（Wait）
- 等待指定秒数
- 用于等待页面加载、动画完成等
- 支持小数点精度（如：0.5秒）

**示例：**
```json
{
  "type": "wait",
  "seconds": 2,
  "label": "等待页面加载"
}
```

## 使用方法

### 在 Web UI 中创建任务

1. 点击"添加任务"
2. 选择"多时间点任务"
3. 添加时间点
4. 在"操作序列"区域点击对应按钮：
   - **+ 点击** - 添加鼠标点击
   - **+ 输入文本** - 添加文本输入
   - **+ 按键** - 添加键盘快捷键
   - **+ 等待** - 添加等待时间

### 操作顺序

操作会按照添加的顺序依次执行，每个操作之间会有设定的间隔时间（默认 0.5 秒）。

## 数据格式

### 新格式（推荐）

使用 `actions` 数组，支持混合操作：

```json
{
  "time": "14:30",
  "name": "任务步骤",
  "show_desktop": true,
  "interval": 0.5,
  "actions": [
    {
      "type": "click",
      "x": 100,
      "y": 200,
      "clicks": 1,
      "label": "点击按钮"
    },
    {
      "type": "type",
      "text": "Hello",
      "interval": 0.05,
      "label": "输入文本"
    },
    {
      "type": "hotkey",
      "keys": ["enter"],
      "label": "回车"
    },
    {
      "type": "wait",
      "seconds": 2,
      "label": "等待"
    }
  ]
}
```

### 旧格式（仍然支持）

使用 `clicks` 数组，仅支持点击：

```json
{
  "time": "14:30",
  "name": "任务步骤",
  "show_desktop": true,
  "interval": 0.5,
  "clicks": [
    [100, 200],      // 单击
    [300, 400, 2],   // 双击
    [500, 600]       // 单击
  ]
}
```

**向后兼容：** 旧任务会继续正常工作，无需修改！

## 技术细节

### 后端改进

1. **Clicker 类扩展**（`src/clicker.py`）
   - 新增 `execute_action()` - 执行单个操作
   - 新增 `execute_actions()` - 执行操作序列
   - 新增 `convert_clicks_to_actions()` - 格式转换

2. **Scheduler 适配**（`src/scheduler.py`）
   - 优先使用新格式 `actions`
   - 自动降级到旧格式 `clicks`（向后兼容）

### 前端改进

1. **操作编辑器**（`static/app.js`）
   - 可视化操作卡片
   - 拖拽排序（未来可扩展）
   - 实时预览

2. **UI 增强**（`templates/index.html`）
   - 操作类型徽章（彩色标识）
   - 操作参数表单
   - 获取鼠标坐标按钮

## 注意事项

⚠️ **输入法问题**
- 文本输入功能仅支持英文字符
- 如需输入中文，请确保输入法处于英文模式
- 或者使用粘贴功能（Ctrl+V）

⚠️ **窗口焦点**
- 键盘操作会发送到当前激活的窗口
- 确保目标窗口在操作时处于前台

⚠️ **安全性**
- 密码等敏感信息会明文保存在配置文件中
- 请妥善保管 `config/tasks.json` 文件
