# ↕️ 操作排序功能使用指南

## ✨ 新增功能：上下移动操作

v0.7.2 新增**操作排序功能**，可以自由调整操作顺序！

---

## 🎯 功能特点

✅ **上移按钮** - 将操作向上移动一位  
✅ **下移按钮** - 将操作向下移动一位  
✅ **即时生效** - 点击即移动，无需保存  
✅ **灵活调整** - 可以将任何操作移到任意位置  

---

## 📖 使用方法

### 按钮说明

每个操作卡片右上角有 3 个按钮：

```
[操作类型]              [↑] [↓] [×]
```

- **↑** - 上移按钮（向上移动一位）
- **↓** - 下移按钮（向下移动一位）
- **×** - 删除按钮（删除此操作）

### 移动操作

**向上移动：**
1. 点击要移动的操作上的 **↑** 按钮
2. 该操作会与上一个操作交换位置
3. 如果已经是第一个，点击无效

**向下移动：**
1. 点击要移动的操作上的 **↓** 按钮
2. 该操作会与下一个操作交换位置
3. 如果已经是最后一个，点击无效

## 💡 使用场景

### 场景 1：在中间插入操作

**原始顺序：**
```
1. [点击] 按钮A
2. [按键] enter
```

**需求：在点击和回车之间插入等待**

**操作步骤：**
1. 点击 **"+ 等待"** 添加等待操作（会加到最后）
2. 点击新添加的等待操作的 **↑** 按钮
3. 继续点击 **↑** 直到移到合适位置

**最终顺序：**
```
1. [点击] 按钮A
2. [等待] 1秒        ← 插入的操作
3. [按键] enter
```

### 场景 2：调整操作顺序

**原始顺序（错误）：**
```
1. [粘贴] 密码       ← 应该在第3步
2. [点击] 用户名框
3. [粘贴] 用户名
4. [按键] tab
```

**调整步骤：**
1. 点击"粘贴密码"的 **↓** 按钮 3 次
2. 将其移到 tab 后面

**最终顺序（正确）：**
```
1. [点击] 用户名框
2. [粘贴] 用户名
3. [按键] tab
4. [粘贴] 密码       ← 已移到正确位置
```

### 场景 3：快速重组

**需要重组多个操作时：**
1. 先删除不需要的操作（点 **×**）
2. 添加新操作（会加到最后）
3. 用 **↑** **↓** 按钮调整到合适位置
4. 比手动删除+重新添加快得多！

## 🔧 技术实现

### JavaScript 函数

```javascript
// 上移操作
function moveActionUp(button) {
    const actionItem = button.closest('.action-item');
    const prevItem = actionItem.previousElementSibling;
    
    if (prevItem && !prevItem.classList.contains('empty-actions')) {
        actionItem.parentNode.insertBefore(actionItem, prevItem);
    }
}

// 下移操作
function moveActionDown(button) {
    const actionItem = button.closest('.action-item');
    const nextItem = actionItem.nextElementSibling;
    
    if (nextItem) {
        actionItem.parentNode.insertBefore(nextItem, actionItem);
    }
}
```

### DOM 操作
- 使用 `insertBefore` 交换元素位置
- 保持操作数据完整性
- 即时更新 UI

## 💡 使用技巧

### 快速插入
1. 先添加所有需要的操作（不用管顺序）
2. 然后用上下按钮调整顺序
3. 比边加边排更快

### 批量调整
- 从下往上调整更快（减少点击次数）
- 例如：要把最后一个移到第一，只需多次点 ↑

### 复制操作
虽然没有复制按钮，但可以：
1. 记住要复制的操作的参数
2. 添加新操作
3. 填入相同参数
4. 用 ↑↓ 移到合适位置

## ⚠️ 注意事项

### 保存提醒
- 移动操作后**记得保存任务**
- 点击"更新任务"或"创建任务"按钮

### 边界情况
- 第一个操作无法上移
- 最后一个操作无法下移
- 只有一个操作时，↑↓ 都无效

### 性能
- 操作数量较多时（50+）移动可能稍慢
- 建议将复杂任务拆分成多个时间点
