## 实现的功能

### 1. 多时间点任务系统 ⭐

**后端实现**：
- `src/config_manager.py` - 支持多任务类型
- `src/scheduler.py` - 多时间点调度逻辑
- `app.py` - REST API 支持

**前端实现**：
- 任务类型切换（简单任务 / 多时间点任务）
- 时间轴可视化编辑器
- 动态添加/删除时间点
- 坐标可视化管理

**特性**：
- 一个任务可配置多个时间点
- 每个时间点独立配置
- 支持不同点击间隔
- 支持回到桌面选项

### 2. Web 可视化编辑器 🎨

**界面功能**：
- 双标签切换（简单任务 / 多时间点）
- 时间点卡片式编辑
- 实时添加/删除坐标
- 一键获取鼠标位置
- 响应式设计（支持手机）

**交互特性**：
- 拖拽友好（虽然不支持拖拽排序）
- Toast 即时反馈
- 表单验证
- 二次确认删除

### 3. 自动回到桌面 🖥️

**实现位置**：`src/clicker.py`

**功能**：
- Windows: 自动按 Win+D
- macOS: 自动按 Fn+F11
- 可选配置（每个时间点独立设置）

## 使用方式

### 方式 1：Web 可视化编辑器（推荐）⭐

```bash
python app.py
```

访问 http://localhost:5000

**操作步骤**：
1. 点击"添加任务"
2. 切换到"多时间点任务"
3. 填写任务名称和描述
4. 点击"+ 添加时间点"
5. 配置每个时间点（名称、时间、坐标、间隔）
6. 点击"获取当前位置"自动获取坐标
7. 点击"创建任务"

### 方式 2：命令行工具

```bash
python create_multi_time_task.py
```

选择模板，自动生成配置。

### 方式 3：手动编辑配置

编辑 `config/tasks.json`

## 完整示例：小程序抢课

### Web 界面操作

1. **启动服务**
   ```bash
   python app.py
   ```

2. **创建任务**
   - 访问 http://localhost:5000
   - 点击"添加任务"
   - 选择"多时间点任务"

3. **配置第一个时间点（20:57 进入小程序）**
   - 点击"+ 添加时间点"
   - 步骤名称：进入小程序
   - 执行时间：20:57
   - ☑ 勾选"执行前回到桌面"
   - 点击间隔：2.0
   - 依次获取 4 个坐标：
     - 移动鼠标到微信图标 → 点击"获取当前位置"
     - 移动鼠标到小程序入口 → 点击"获取当前位置"
     - 移动鼠标到约课小程序 → 点击"获取当前位置"
     - 移动鼠标到约课页面 → 点击"获取当前位置"

4. **配置第二个时间点（21:00 刷新抢课）**
   - 再次点击"+ 添加时间点"
   - 步骤名称：刷新抢课
   - 执行时间：21:00
   - 点击间隔：0.3
   - 依次获取 3 个坐标：
     - 刷新按钮
     - 目标课程
     - 确认按钮

5. **创建并启动**
   - 点击"创建任务"
   - 点击"启动调度器"
   - 等待自动执行

## 技术架构

### 前端（Web）

```
templates/index.html    - HTML 结构 + CSS 样式
static/app.js          - JavaScript 逻辑

功能模块：
- 任务类型切换
- 时间轴编辑器
- 坐标管理
- API 调用
- 状态同步
```

### 后端（Flask + Python）

```
app.py                 - Flask API 服务
src/config_manager.py  - 配置管理
src/scheduler.py       - 任务调度
src/clicker.py         - 点击执行

API 接口：
- GET  /api/tasks              - 获取任务列表
- POST /api/tasks              - 创建任务（支持多类型）
- DELETE /api/tasks/<id>       - 删除任务
- POST /api/tasks/<id>/toggle  - 启用/禁用
- GET  /api/mouse/position     - 获取鼠标位置
- POST /api/scheduler/start    - 启动调度器
- POST /api/scheduler/stop     - 停止调度器
- GET  /api/scheduler/status   - 获取状态
```
