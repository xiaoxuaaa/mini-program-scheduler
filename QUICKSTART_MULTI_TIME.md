# 快速开始 - 多时间点任务系统

## 🚀 30秒快速上手

### 场景：小程序抢课

你的需求：
- **20:57** - 进入微信小程序
- **21:00** - 刷新并抢课

---

## 步骤 1：获取所有坐标 (5分钟)

### 方法 A：使用 Web 界面
```bash
python app.py
# 访问 http://localhost:5000
# 点击"添加任务" → "获取当前位置"
```

### 方法 B：使用命令行
```bash
python main.py
# 选择 5. 获取鼠标坐标
```

### 需要获取的坐标

**第一阶段（20:57 进入小程序）**：
1. 桌面微信图标位置
2. 小程序入口按钮
3. 你的小程序图标
4. 约课页面入口

**第二阶段（21:00 抢课）**：
5. 刷新按钮
6. 目标课程位置
7. 确认按钮

---

## 步骤 2：创建任务 (1分钟)

运行创建工具：
```bash
python create_multi_time_task.py
```

选择 **1. 创建小程序抢课任务**

---

## 步骤 3：修改坐标 (3分钟)

打开 `config/tasks.json`，找到刚创建的任务：

```json
{
  "id": 1,
  "name": "小程序抢课",
  "type": "multi_time",
  "timeline": [
    {
      "time": "20:57",
      "name": "进入小程序",
      "show_desktop": true,
      "clicks": [
        [100, 600],   // ← 替换：桌面微信图标
        [500, 300],   // ← 替换：小程序入口
        [600, 400],   // ← 替换：你的小程序
        [700, 500]    // ← 替换：约课页面
      ],
      "interval": 2.0
    },
    {
      "time": "21:00",
      "name": "刷新抢课",
      "clicks": [
        [800, 300],   // ← 替换：刷新按钮
        [850, 400],   // ← 替换：目标课程
        [900, 500]    // ← 替换：确认按钮
      ],
      "interval": 0.3
    }
  ]
}
```

将 `[x, y]` 替换为你的实际坐标。

---

## 步骤 4：测试运行 (5分钟)

### 测试方法 1：修改时间测试

将时间改为当前时间后 2 分钟：
```json
"time": "18:35"  // 如果现在是 18:33
```

### 测试方法 2：分步测试

创建两个简单任务，分别测试每个阶段。

---

## 步骤 5：启动调度 (1分钟)

```bash
python app.py
```

访问 http://localhost:5000

点击 **"启动调度器"**

---

## 步骤 6：等待执行

✅ **20:57** - 系统会自动：
- 回到桌面
- 点击微信
- 进入小程序
- 打开约课页面

✅ **21:00** - 系统会自动：
- 点击刷新（0.3秒）
- 点击课程（0.3秒）
- 点击确认

## 💡 调试技巧

### 坐标不准确？

**方法 1**：截图标记法
1. 截图目标位置
2. 用画图工具查看坐标
3. 微调 ±10 像素

**方法 2**：实时测试法
1. 创建 1 分钟后的测试任务
2. 观察点击位置
3. 调整坐标后重试

### 时间来不及？

调整间隔时间：
```json
"interval": 1.5  // 缩短到 1.5 秒
```

### 第一步失败了？

每个时间点独立，不影响后续执行。

---

## ⚠️ 注意事项

### 执行前确认

1. **屏幕分辨率**：确保和获取坐标时一致
2. **窗口位置**：微信位置不要改变
3. **电脑状态**：不要锁屏或休眠
4. **网络连接**：确保网络正常

### 最佳实践

1. **提前测试**：至少提前 1 天测试一遍
2. **备份方案**：准备手动操作的备份
3. **多次点击**：重要按钮可以点 2 次确保成功
4. **时间预留**：预留 1-2 秒缓冲时间

## 📞 常见问题

**Q: 需要一直开着浏览器吗？**

A: 不需要。启动调度器后可以关闭浏览器，但**不能关闭命令行窗口**。

**Q: 可以最小化窗口吗？**

A: 可以！调度器在后台运行。

**Q: 抢课成功率高吗？**

A: 取决于：
- 坐标准确性（90%+）
- 网络延迟
- 竞争人数

建议：
- 提前测试确保流程正确
- 间隔设为 0.2-0.3 秒
- 重要按钮可以点 2 次

**Q: 失败了怎么办？**

A: 
1. 查看命令行输出日志
2. 检查坐标是否准确
3. 手动再试一次
