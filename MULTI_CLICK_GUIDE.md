# 多次点击功能说明

## ✅ 完全支持！

我们的程序从一开始就支持在一个任务中配置**多个点击坐标**，会按顺序执行。

## 🎯 使用场景

### 场景 1: 小程序抢购（多步操作）

```
步骤1: 点击商品 → 进入详情页
步骤2: 点击"立即购买" → 进入订单页
步骤3: 点击"确认订单" → 完成购买
```

**配置方式**：
```
任务名称: 商品抢购
时间: 14:00
坐标 1: 640,400  ← 商品位置
坐标 2: 640,800  ← 立即购买按钮
坐标 3: 640,900  ← 确认订单按钮
坐标 4: done
```

### 场景 2: 游戏连续操作

```
步骤1: 点击"每日任务"
步骤2: 点击"领取奖励"
步骤3: 点击"关闭"
```

**配置方式**：
```
任务名称: 每日任务领取
时间: 09:00
坐标 1: 2        ← 使用自动获取（移到"每日任务"）
坐标 2: 2        ← 使用自动获取（移到"领取奖励"）
坐标 3: 2        ← 使用自动获取（移到"关闭"）
坐标 4: done
```

### 场景 3: 重复点击同一位置

```
场景: 某些操作需要连续点击同一按钮多次
例如: 游戏中的"跳过"按钮可能需要点击3次才能完全跳过
```

**配置方式**：
```
任务名称: 跳过动画
时间: 10:00
坐标 1: 900,800  ← 跳过按钮
坐标 2: 900,800  ← 同一位置再点一次
坐标 3: 900,800  ← 第三次点击
坐标 4: done
```

## ⚙️ 技术参数

### 点击间隔

默认每次点击之间间隔 **0.5 秒**：

```python
# 在 src/clicker.py 中
def __init__(self, click_interval: float = 0.5):
    self.click_interval = click_interval
```

**执行示例**：
```
开始执行点击序列，共 3 个点击
  [1/3] 点击坐标 (640, 400)
  # 等待 0.5 秒
  [2/3] 点击坐标 (640, 800)
  # 等待 0.5 秒
  [3/3] 点击坐标 (640, 900)
✓ 任务执行成功
```

### 支持的点击数量

理论上**无限制**，但建议：
- ✅ 3-5 个点击：最常见，推荐
- ✅ 5-10 个点击：复杂流程
- ⚠️ 10+ 个点击：确保每步都正确，建议分拆为多个任务

## 💡 实用技巧

### 技巧 1: 按顺序获取坐标

```
1. 打开目标应用
2. 导航到起始页面
3. 添加任务，逐个获取坐标
4. 按操作流程顺序获取：
   第1个坐标 → 第1步操作位置
   第2个坐标 → 第2步操作位置
   第3个坐标 → 第3步操作位置
```

### 技巧 2: 混合输入方式

```
坐标 1: 2          ← 自动获取
坐标 2: 640,800    ← 直接输入（已知坐标）
坐标 3: 1          ← 手动输入（需要确认）
  请输入坐标 (x,y): 640,900
坐标 4: done
```

### 技巧 3: 记录坐标说明

在任务描述中记录每个点击的含义：

```
任务描述: 1.点商品 2.立即购买 3.确认订单
```

这样查看任务时能快速回忆每个坐标的作用。

### 技巧 4: 测试单个步骤

如果不确定多步操作是否准确：

```
1. 先创建单步测试任务
   任务1: 只点第1步
   任务2: 只点第2步
   任务3: 只点第3步

2. 分别测试每个步骤

3. 确认都准确后，创建完整任务
   任务4: 包含所有步骤
```

## ⚠️ 注意事项

### 1. 页面加载时间

如果操作之间需要等待页面加载：

**当前**：默认 0.5 秒间隔
```
点击1 → 等0.5秒 → 点击2 → 等0.5秒 → 点击3
```

**如果页面加载慢**：
- 方案A: 手动修改间隔时间（需要编辑代码）
- 方案B: 创建多个任务，设置不同时间
  ```
  任务1: 14:00:00 - 点击第1步
  任务2: 14:00:02 - 点击第2步（2秒后）
  任务3: 14:00:04 - 点击第3步（4秒后）
  ```

### 2. 坐标准确性

多步操作对坐标准确性要求更高：
- ✅ 逐步测试每个坐标
- ✅ 确保窗口位置固定
- ✅ 第一次使用建议设置测试任务

### 3. 失败处理

如果某一步失败：
- 程序会继续执行后续步骤
- 最终显示"任务执行失败"
- 建议查看执行日志确认哪步失败
