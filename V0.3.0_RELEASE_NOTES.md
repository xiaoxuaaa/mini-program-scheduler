# v0.3.0 更新说明

## 🎯 核心变更

### 移除微信窗口依赖

**之前的版本 (v0.2.0)**：
```
启动调度器 → 到达时间 → 查找微信窗口 → 激活窗口 → 执行点击
```

**现在的版本 (v0.3.0)**：
```
启动调度器 → 到达时间 → 直接执行点击
```

### 主要改动

✅ **移除的功能**：
- 微信窗口查找逻辑
- 窗口激活步骤
- window_helper 依赖

✅ **保留的功能**：
- 定时任务调度
- 坐标点击
- 任务管理
- 坐标获取工具

## 📊 优势对比

| 特性 | v0.2.0（微信专用） | v0.3.0（通用工具） |
|------|-------------------|-------------------|
| 应用范围 | 仅微信小程序 | 任何应用 |
| 执行速度 | 慢（需查找+激活窗口） | 快（直接点击） |
| 使用限制 | 必须打开微信 | 无特定应用限制 |
| 适用场景 | 微信小程序操作 | 通用自动化 |
| 代码复杂度 | 高 | 低 |

## ⚠️ 使用注意事项

### 重要变化

1. **不再自动切换窗口**
   - 之前：自动查找并激活微信窗口
   - 现在：直接点击屏幕坐标
   - **要求**：目标应用窗口需要在前台或该坐标位置可点击

2. **适用任何应用**
   - 不再局限于微信
   - 可以用于任何需要定时点击的场景
   - 更加灵活

3. **执行速度更快**
   - 无需查找窗口（省约 0.5-1 秒）
   - 无需激活窗口（省约 0.5 秒）
   - 到点即刻执行

### 使用建议

#### ✅ 最佳实践

1. **提前准备窗口**
   ```
   - 打开目标应用
   - 导航到目标页面
   - 保持窗口在前台
   - 确保坐标位置可见
   ```

2. **固定窗口位置**
   ```
   - 固定应用窗口大小
   - 固定应用窗口位置
   - 避免移动或调整窗口
   ```

3. **测试验证**
   ```
   - 首次使用设置测试任务
   - 设置2-3分钟后执行
   - 观察点击位置是否准确
   ```

#### ❌ 注意避免

1. **不要最小化目标窗口**
   - 最小化后坐标可能不可点击
   - 建议保持窗口可见

2. **不要遮挡点击位置**
   - 确保坐标位置没有被其他窗口遮挡
   - 如果有弹窗，可能点击失效

3. **不要频繁调整窗口**
   - 窗口大小改变会导致坐标偏移
   - 需要重新获取坐标

## 🔧 技术细节

### 代码变更

**scheduler.py 简化**：
```python
# 移除前（v0.2.0）
def _execute_task(self, task):
    # 1. 查找微信窗口
    hwnd = self.window_helper.find_wechat_window()
    if not hwnd:
        return
    
    # 2. 激活窗口
    self.window_helper.activate_window(hwnd)
    time.sleep(0.5)
    
    # 3. 执行点击
    self.clicker.click_sequence(clicks)

# 移除后（v0.3.0）
def _execute_task(self, task):
    # 直接执行点击
    time.sleep(0.5)  # 短暂准备
    self.clicker.click_sequence(clicks)
```

- pywin32 - window_helper 已移除，但保留依赖以防未来扩展
