# 提高调度器精度说明

## 当前精度

**检查间隔**: 1秒  
**延迟范围**: 0-2秒  
**适用场景**: 绝大多数场景（抢购、签到等）

## 如何提高精度？

### 方法：修改检查间隔

**文件**: `src/scheduler.py`  
**位置**: 第114行

```python
# 当前代码
def _run_scheduler(self):
    while self.running:
        schedule.run_pending()
        time.sleep(1)  # ← 每秒检查一次
```

**修改为**：

```python
# 提高精度到0.1秒
def _run_scheduler(self):
    while self.running:
        schedule.run_pending()
        time.sleep(0.1)  # ← 每0.1秒检查一次
```

### 精度对比

| 检查间隔 | 延迟范围 | CPU占用 | 适用场景 |
|---------|---------|---------|---------|
| 1秒（默认） | 0-2秒 | 极低 | 日常使用 ✅ |
| 0.5秒 | 0-1秒 | 低 | 对时间要求较高 |
| 0.1秒 | 0-0.2秒 | 中 | 精确抢购（毫秒级） |
| 0.01秒 | 0-0.02秒 | 高 | 极端场景（不推荐） |

## 实际影响分析

### 场景1: 小程序抢购（14:00开售）

**1秒间隔（默认）**：
```
14:00:01.xxx 执行 - 比其他人慢1秒
```

**0.1秒间隔**：
```
14:00:00.1xx 执行 - 几乎准时
```

### 场景2: 游戏签到（09:00）

**差异不大**：
- 签到通常不需要毫秒级精度
- 1-2秒延迟不影响

### 场景3: 秒杀抢购（20:00:00开售）

**关键场景**：
- 0.1秒间隔更有优势
- 建议使用高精度模式

## 注意事项

### CPU占用

```
检查频率越高 → CPU唤醒越频繁 → 耗电越多
```

**建议**：
- 笔记本电池供电：使用1秒或0.5秒
- 台式机/充电状态：可以使用0.1秒

### 实际测试

修改后建议测试：
```
1. 设置一个2分钟后的测试任务
2. 观察实际执行时间
3. 确认延迟是否满足需求
```
