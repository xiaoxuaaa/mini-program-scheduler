# 使用演示 - 坐标自动获取功能

## 🎯 场景演示：设置小程序商品抢购任务

假设你需要在下午 2:00 准时抢购一个限量商品。

### 步骤 1: 启动程序

```bash
python main.py
```

### 步骤 2: 准备微信小程序页面

在启动程序后，打开微信：
1. 进入目标小程序
2. 找到商品页面
3. 确保"立即购买"按钮可见
4. **保持微信窗口打开**（可以最小化）

### 步骤 3: 添加任务（使用新功能）

在主菜单中选择 `1`，然后按照提示操作：

```
=== 添加新任务 ===

任务名称: 限量商品抢购
任务描述（可选）: XX品牌限量款
执行时间（格式 HH:MM，如 14:30）: 14:00

输入点击坐标（可多个）
选项:
  1. 手动输入坐标（格式: x,y）
  2. 使用工具获取坐标（自动获取）
  输入 'done' 完成坐标输入

坐标 1 (输入1/2或坐标或done): 2
```

👉 **此时输入 `2` 后会发生什么：**

```
  >> 移动鼠标到目标位置，3秒倒计时后自动获取...
     3...
```

⚡ **立即行动**：
- 快速切换到微信窗口（Alt+Tab）
- 将鼠标移动到"立即购买"按钮上
- 保持不动

```
     2...
     1...
  ✓ 已自动获取坐标 (720, 850)
```

🎉 **成功**！坐标已自动获取。

### 步骤 4: 添加更多点击（如需要）

如果需要多次点击（例如：点击商品 → 立即购买 → 确认订单），继续添加：

```
坐标 2 (输入1/2或坐标或done): 2
  >> 移动鼠标到目标位置，3秒倒计时后自动获取...
     3...
     2...
     1...
  ✓ 已自动获取坐标 (640, 920)

坐标 3 (输入1/2或坐标或done): done
```

### 步骤 5: 确认任务

```
✓ 任务添加成功！
  名称: 限量商品抢购
  时间: 14:00
  点击数: 2
```

### 步骤 6: 查看任务（可选）

在主菜单选择 `2` 查看所有任务，确认信息正确。

### 步骤 7: 启动调度器

在主菜单选择 `6`：

```
调度器已启动，等待任务执行...
提示: 按 Ctrl+C 停止
当前时间: 2026-06-16 13:45:30

待执行任务:
  - 下次执行: 2026-06-16 14:00:00
```

### 步骤 8: 等待自动执行

- 保持程序运行
- 保持微信打开（可以最小化）
- 到 14:00 时，程序会自动执行点击操作

## 💡 实用技巧

### 技巧 1: 测试坐标准确性

在正式任务前先测试：

```
任务名称: 测试任务
时间: 13:47  ← 设置为2分钟后
坐标 1: 输入 2，获取坐标
done
```

启动调度器，观察2分钟后是否点击准确。

### 技巧 2: 多屏幕用户

如果你有两个显示器：
1. 一个屏幕显示命令行
2. 另一个屏幕显示微信
3. 输入 `2` 后，视线移到另一个屏幕，移动鼠标

### 技巧 3: 单屏幕用户

1. 输入 `2` 后立即 Alt+Tab 切换到微信
2. 快速移动鼠标到目标按钮
3. 3秒时间通常足够完成切换和定位

### 技巧 4: 复杂流程规划

**场景**：需要点击多个位置

在添加任务前，先在纸上或脑海中规划顺序：
1. 点击商品进入详情页 → 坐标1
2. 滚动后点击"立即购买" → 坐标2
3. 点击"确认订单" → 坐标3

然后按顺序逐个获取坐标。

### 技巧 5: 如果获取错了坐标

不要担心！可以：
1. 继续完成任务添加
2. 然后删除这个任务（菜单选项 3）
3. 重新添加正确的任务

或者，在获取坐标时如果发现错了，可以先添加占位坐标（如 0,0），完成后删除任务重建。

## ⚠️ 常见问题

### Q1: 3秒不够用怎么办？

**解决方案**：
- 提前准备好微信窗口，放在容易切换的位置
- 或者使用方式1（手动输入），先用菜单5单独获取坐标再手动输入

### Q2: 坐标获取不准确

**可能原因**：
- 窗口大小改变了
- 屏幕分辨率改变了
- Windows 缩放比例改变了

**解决方案**：
- 固定微信窗口大小
- 或重新获取坐标

### Q3: 倒计时时鼠标动了

**解决方案**：
- 最后捕获的位置是最终位置，确保倒计时结束前停在目标位置即可
- 或者重新获取这个坐标（删除任务重建，或继续添加正确的坐标）
