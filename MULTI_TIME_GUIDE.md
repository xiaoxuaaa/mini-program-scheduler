# 多时间点任务系统 - 使用指南

## 🎉 新功能概览

v0.6.0 版本已支持**多时间点任务**，可以在一个任务中配置多个时间点，每个时间点执行不同的操作。

---

## 核心特性

### 1. 多时间点调度
- 一个任务可包含多个时间点
- 每个时间点独立执行
- 自动按时间顺序触发

### 2. 独立配置
每个时间点可以配置：
- ✅ **执行时间** - 精确到分钟
- ✅ **步骤名称** - 方便识别
- ✅ **点击坐标** - 支持多个坐标
- ✅ **点击间隔** - 自定义间隔时间
- ✅ **回到桌面** - 执行前自动回到桌面

### 3. 自动回到桌面
- 支持 `show_desktop` 选项
- 按 `Win+D` 快捷键回到桌面
- 适合从桌面开始的操作流程

---

## 使用场景示例

### 场景：小程序抢课

**需求**：
- 20:57 进入微信小程序
- 21:00 刷新页面并抢课

**解决方案**：创建多时间点任务

```json
{
  "name": "小程序抢课",
  "type": "multi_time",
  "timeline": [
    {
      "time": "20:57",
      "name": "进入小程序",
      "show_desktop": true,
      "clicks": [
        [100, 600],  // 点击微信图标
        [500, 300],  // 点击小程序
        [600, 400]   // 进入约课页面
      ],
      "interval": 2.0
    },
    {
      "time": "21:00",
      "name": "刷新抢课",
      "clicks": [
        [800, 300],  // 刷新
        [850, 400],  // 选课
        [900, 500]   // 确认
      ],
      "interval": 0.3
    }
  ]
}
```

## 创建方式

### 方式 1：使用脚本工具（推荐）⭐

**步骤**：

1. 运行创建工具：
```bash
python create_multi_time_task.py
```

2. 选择"创建小程序抢课任务"

3. 编辑生成的任务配置（修改坐标）

4. 启动 Web 服务查看任务：
```bash
python app.py
```

---

### 方式 2：直接编辑配置文件

1. 打开 `config/tasks.json`

2. 添加任务配置：

```json
{
  "id": 1,
  "name": "我的多时间点任务",
  "type": "multi_time",
  "description": "任务描述",
  "enabled": true,
  "created_at": "2026-06-23T18:00:00",
  "last_run": null,
  "timeline": [
    {
      "time": "14:30",
      "name": "第一步",
      "show_desktop": false,
      "clicks": [[100, 200], [300, 400]],
      "interval": 1.0
    },
    {
      "time": "14:35",
      "name": "第二步",
      "show_desktop": false,
      "clicks": [[500, 600]],
      "interval": 0.5
    }
  ]
}
```

### 方式 3：通过 API

curl -X POST http://localhost:5000/api/tasks \

  -d '{

        "name": "步骤1",
        "clicks": [[100, 200]],

  }'

## 配置说明

### Timeline 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `time` | string | ✅ | 执行时间，格式 `HH:MM` |
| `name` | string | ✅ | 步骤名称 |
| `clicks` | array | ✅ | 点击坐标列表 `[[x1,y1], [x2,y2]]` |
| `interval` | float | ❌ | 点击间隔（秒），默认 0.5 |
| `show_desktop` | boolean | ❌ | 是否先回到桌面，默认 false |

### 任务类型

- `"simple"` - 简单任务（单一时间点）
- `"multi_time"` - 多时间点任务

## 常见问题

### Q: 如何测试坐标是否正确？

A: 两种方法：
1. 修改时间为 1 分钟后，测试执行
2. 临时创建简单任务，单独测试每个步骤

### Q: 第一步失败了，第二步还会执行吗？

A: 会的。每个时间点独立执行，不受前面步骤影响。

### Q: 可以有多少个时间点？

A: 理论上没有限制，但建议不超过 10 个。

### Q: 间隔时间如何设置？

A: 
- 一般操作：1-2 秒
- 抢购场景：0.2-0.3 秒
- 等待加载：2-3 秒

### Q: `show_desktop` 什么时候用？

A: 当操作从桌面开始时使用，比如点击桌面图标。

## 兼容性

### 旧版任务

- ✅ 完全兼容
- 旧的简单任务自动识别为 `type: "simple"`
- 无需修改现有任务

### 升级路径

简单任务 → 多时间点任务：

**之前**（两个独立任务）：
```json
{"id": 1, "time": "20:57", "clicks": [...]}
{"id": 2, "time": "21:00", "clicks": [...]}
```

**之后**（一个多时间点任务）：
```json
{
  "id": 1,
  "type": "multi_time",
  "timeline": [
    {"time": "20:57", "clicks": [...]},
    {"time": "21:00", "clicks": [...]}
  ]
}
```
