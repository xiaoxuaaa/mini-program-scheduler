# v0.4.0 更新说明 - 后台运行模式

## 🎉 重大改进

### 问题描述

**v0.3.0 的问题**：
```
启动调度器（菜单6）→ 调度器运行 → 界面卡住 → 无法输入其他操作 → 只能关闭CMD
```

这是因为调度器在主线程中运行，阻塞了整个程序。

### 解决方案

**v0.4.0 引入后台运行**：
```
启动调度器（菜单6）→ 调度器在后台运行 → 界面正常 → 可以继续操作菜单
```

## ✨ 新增功能

### 1. 后台运行调度器

调度器现在使用独立线程运行，不再阻塞主界面：


随时可以停止运行中的调度器：

```

✓ 调度器已停止
```

实时查看调度器运行状态：

### 4. 智能退出

退出程序时自动停止调度器：

## 🔧 技术实现

### 多线程架构

```python
# 主线程：处理用户输入和菜单操作
main_thread: 
    - 显示菜单
    - 处理用户选择
    - 管理任务

# 后台线程：运行调度器
scheduler_thread (daemon):
    - 检查时间
    - 执行任务
    - 更新状态
```

### 线程安全

- 使用 `daemon=True`，主程序退出时自动结束
- 停止时使用 `thread.join(timeout=2)` 优雅退出
- 通过 `self.running` 标志控制线程生命周期

## ⚠️ 注意事项

### 1. 调度器状态持久化

调度器状态是内存中的，程序关闭后不保存：
```
✓ 任务配置保存（config/tasks.json）
✗ 调度器状态不保存（重启需要重新启动）
```

### 2. 后台运行提示

调度器在后台运行时：
- ✅ 可以继续操作菜单
- ✅ 到点会自动执行任务
- ✅ 任务执行信息会显示在命令行
- ⚠️ 不要关闭命令行窗口

### 3. 任务动态加载

```
启动调度器后添加的新任务不会自动加载
解决方案：
  1. 停止调度器（菜单 7）
  2. 重新启动调度器（菜单 6）
```
