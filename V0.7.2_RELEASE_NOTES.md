## ✨ 新增功能

### 🔼🔽 操作排序

#### 功能特性
- ✅ **上移按钮** - 将操作向上移动一位
- ✅ **下移按钮** - 将操作向下移动一位
- ✅ **任意位置插入** - 可以在操作序列中任意位置插入新操作
- ✅ **实时预览** - 排序后立即显示新顺序

#### 使用场景
- **调整操作顺序** - 发现顺序不对时快速调整
- **插入遗漏操作** - 在中间插入忘记添加的步骤
- **优化任务流程** - 反复调试找到最优顺序

#### 操作方式
```
每个操作卡片右上角：
┌─────────────────────┐
│ 操作类型   ↑ ↓ ✕   │  ← 上移 下移 删除
│ 参数详情            │
└─────────────────────┘
```

## 🔧 技术改进

### 前端增强 (`static/app.js`)

#### 新增排序函数
```javascript
// 上移操作
function moveActionUp(index) {
    if (index > 0) {
        [actions[index], actions[index-1]] = 
        [actions[index-1], actions[index]];
        renderActions();
    }
}

// 下移操作
function moveActionDown(index) {
    if (index < actions.length - 1) {
        [actions[index], actions[index+1]] = 
        [actions[index+1], actions[index]];
        renderActions();
    }
}
```

#### 智能按钮禁用
- 第一个操作：上移按钮禁用
- 最后一个操作：下移按钮禁用

### UI 优化 (`templates/index.html`)

#### 操作按钮分组
```html
<div class="action-buttons">
    <button class="btn-small" onclick="moveActionUp(i)">↑</button>
    <button class="btn-small" onclick="moveActionDown(i)">↓</button>
    <button class="btn-small btn-danger" onclick="deleteAction(i)">✕</button>
</div>
```

#### 样式增强
- 按钮分组布局
- 悬停效果优化
- 禁用状态样式
- 图标大小统一

## 💡 使用示例

### 场景 1: 调整操作顺序

**原始顺序（错误）:**
```
1. 点击"提交"按钮
2. 点击"用户名"输入框
3. 输入用户名
```

**调整后（正确）:**
```
1. 点击"用户名"输入框  ← 从第2位上移
2. 输入用户名            ← 从第3位上移
3. 点击"提交"按钮
```

### 场景 2: 插入遗漏操作

**原始:**
```
1. 点击"用户名"输入框
2. 输入用户名
3. 点击"提交"按钮      ← 缺少密码输入！
```

**添加并排序:**
```
1. 点击"用户名"输入框
2. 输入用户名
3. 按 Tab 键             ← 新增
4. 输入密码             ← 新增，然后上移到位
5. 点击"提交"按钮
```
